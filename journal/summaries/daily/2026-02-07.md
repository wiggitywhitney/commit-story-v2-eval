## Key Decisions

- **Restore Rich Library Requirement Objects**: Expanded library requirement tracking to include package and import names, enabling more deterministic SDK initialization and richer context for library discovery.

- **Priority-Based Instrumentation Hierarchy**: Replaced hardcoded span caps with a 4-tier priority system and introduced a ~20% function instrumentation backstop to provide more flexible instrumentation control.

- **Configurable Review Sensitivity**: Replaced hardcoded span run limits with three review modes (strict, moderate, off) to control PR summary annotations.

- **Dry Run Semantics**: Agents run normally and write files, but the Coordinator reverts all file changes from snapshots instead of committing them, preserving results while preventing persistent filesystem modifications.

- **Add Total Functions Tracking**: Added `total_functions_in_file` field to enable ratio-based backstop calculation and provide the denominator for function instrumentation analysis.

- **Implement File Exclude Patterns**: Added configurable glob patterns to skip file processing, with automatic exclusion of SDK init and test files by default.

- **Add Per-File Agent Notes**: Created optional `notes` field in FileResult interface to allow agents to explain judgment calls and provide context.

- **Schema Hash and Agent Version Tracking**: Added `schemaHashBefore`, `schemaHashAfter`, and `agentVersion` fields to support debugging and result comparison.

- **Move CLI and GitHub Action to PoC Scope**: Promoted CLI and GitHub Action from future scope to current proof-of-concept scope with defined architectural constraints.

- **Add Progress Callbacks**: Defined `CoordinatorCallbacks` interface with `onValidationComplete` callback for interface-agnostic progress reporting.

- **Config Validation with Zod**: Implemented runtime config validation mechanism to catch configuration errors early with clear messages.

- **Integrate Weaver Registry Diff Command**: Added registry diff command to track schema changes across multiple workflow stages, supporting programmatic detection of schema extensions and enforcement of "extend only" modification rules.

- **Add Research Phase to Testing Infrastructure PRD**: Made testing infrastructure PRD global in scope with an explicit research phase to investigate best practices for guardrails, testing, and CI/CD patterns.

- **Replace Get Shit Done Workflow with Claude Autonomous Mode**: Updated workflow experiment to use Claude with `--dangerously-skip-permissions` and compare it against a human-supervised PRD system using the same approved work chunks.
