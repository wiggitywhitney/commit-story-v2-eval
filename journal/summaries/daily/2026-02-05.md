## Key Decisions

- **Testing Infrastructure for Autonomous Development**: Use Vitest as the primary testing framework with 80% coverage threshold, configure hooks in `.claude/settings.json` for post-edit type checking and linting, and break implementation into six milestones.

- **System Noise Filtering**: Filter out messages starting with '<' or '[Request interrupted' to catch XML-style system tags and user interrupts, eliminating the need to maintain a specific list of noise patterns.

- **Dynamic Implementation Guidance for Technical Decisions**: Analyze commit content dynamically to categorize files as documentation vs functional code, then generate specific guidance for the AI about what constitutes an "Implemented" vs "Discussed" decision based on actual files changed.

- **Consolidate Message Filtering Logic**: Move substantial chat detection entirely into the message filter module to avoid multiple passes over messages and centralize all message quality analysis in one place.

- **Remove Content Verification from Technical Decisions**: Trust the AI's extraction process and rely only on Zod schema validation for structure, since word-matching verification was too strict and filtered out valid decisions when the AI paraphrased reasoning.

- **Two-Agent Telemetry Architecture**: Use a Coordinator (deterministic script) for mechanical tasks like branch management and file iteration, and an Instrumentation Agent for intelligent code transformation.

- **Schema-Driven Instrumentation Strategy**: Prioritize auto-instrumentation libraries over manual spans, check the existing schema for library references, and add manual spans only for business logic not covered by libraries.

- **Mandatory Initialization Phase**: Verify project prerequisites, validate an existing Weaver schema, create a configuration file, and extract project namespace from package.json before instrumentation begins.

- **Fresh AI Instance Per File**: Create a new invocation of the instrumentation agent for each file to ensure full AI attention, rather than trying to maintain system-wide context across files.

- **Partial Success Over Rollback**: Commit successfully instrumented files, skip failed ones, and create a PR with a summary noting what failed and why, rather than rolling back partial work which adds complexity without learning value.
